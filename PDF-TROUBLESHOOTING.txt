================================================================================
COMMON PROBLEMS & SOLUTIONS
================================================================================

PROBLEM 1: "Libraries not loaded" error
---------------------------------------
CAUSE: Internet connection issue or CDN blocked

SOLUTIONS:
✓ Solution A: Check internet connection
  - Make sure you're connected to the internet
  - Try opening google.com in a new tab
  - If blocked, proceed to Solution B

✓ Solution B: Use Print-to-PDF instead
  - On the report page, click "🖨️ Print Report" button
  - In the print dialog, select "Save as PDF" or "Microsoft Print to PDF"
  - Choose where to save
  - Click Save
  - This ALWAYS works, no internet needed

✓ Solution C: Refresh the page
  - Press Ctrl + F5 (Windows) or Cmd + Shift + R (Mac)
  - This forces reload of libraries
  - Try PDF download again

PROBLEM 2: PDF button does nothing
----------------------------------
CAUSE: JavaScript error or libraries not loaded

SOLUTIONS:
✓ Check browser console
  - Press F12 to open developer tools
  - Click "Console" tab
  - Look for red error messages
  - Send me the error if you need help

✓ Try different browser
  - Chrome (BEST - recommended)
  - Edge (Good)
  - Firefox (Good)
  - Safari (May have issues)

✓ Disable browser extensions
  - Ad blockers can block CDN libraries
  - Try in Incognito/Private mode (Ctrl + Shift + N)

PROBLEM 3: PDF downloads but is blank
-------------------------------------
CAUSE: Timing issue or rendering problem

SOLUTIONS:
✓ Wait longer
  - After clicking, wait 10-15 seconds
  - Don't click anything else
  - Watch for success message

✓ Scroll to top of page
  - Make sure report is scrolled to the top
  - Then click PDF button

✓ Use Print method instead
  - More reliable for complex reports
  - Click "Print Report" button
  - Save as PDF from print dialog

PROBLEM 4: "html2canvas is not defined" error
---------------------------------------------
CAUSE: Library didn't load from CDN

SOLUTIONS:
✓ Check internet connection
  - Must be online for first load
  - Libraries load from CloudFlare CDN

✓ Check firewall/antivirus
  - May be blocking cdnjs.cloudflare.com
  - Temporarily disable and try again
  - Or use Print-to-PDF method

✓ Use alternative download method
  - We've built in automatic fallback
  - If PDF fails, it will ask if you want to use Print instead
  - Click "OK" to use Print method

PROBLEM 5: Filename is wrong or generic
---------------------------------------
CAUSE: Person name field was empty or had special characters

SOLUTION:
✓ Make sure you entered a name in the form
  - The filename uses the person's name
  - Example: VitalMetrics_Report_John_Smith_2026-07-05.pdf
  - Special characters are replaced with underscores

PROBLEM 6: PDF quality is poor
------------------------------
CAUSE: Low resolution or compression

SOLUTION:
✓ Current settings are already optimized (2x scale, 95% quality)
✓ Use Print-to-PDF for highest quality
  - Click Print button
  - Select "Save as PDF"
  - This gives native vector quality (best possible)

PROBLEM 7: Can't find downloaded PDF
------------------------------------
SOLUTIONS:
✓ Check Downloads folder
  - Windows: C:\Users\[YourName]\Downloads
  - Or press Windows + E, click "Downloads"

✓ Check browser's download bar
  - Look at bottom of browser window
  - Or click download icon (usually top-right)
  - Click "Show in folder"

✓ Check browser settings
  - Browser may be asking where to save
  - Check if download was blocked
  - Enable automatic downloads in settings

PROBLEM 8: "Network error" or "Failed to load"
----------------------------------------------
CAUSE: CDN connection issue

SOLUTIONS:
✓ Check your network
  - Make sure internet is working
  - Try opening https://cdnjs.cloudflare.com in browser
  - If blocked, contact IT department

✓ Use offline method
  - The Print-to-PDF method works without internet
  - Click "Print Report" → Save as PDF

✓ Try mobile hotspot
  - If corporate network blocks CDN
  - Use phone's internet temporarily
  - Load the page once
  - Libraries will be cached

================================================================================
DEVELOPER NOTES (FOR DEBUGGING)
================================================================================

If you see errors in console (F12):

Error: "html2canvas is not defined"
→ Library didn't load, check network tab

Error: "jspdf is not defined"
→ Library didn't load, check network tab

Error: "Cannot read property 'jsPDF' of undefined"
→ jsPDF loaded incorrectly, try refreshing

Error: "Failed to execute 'toDataURL' on 'HTMLCanvasElement'"
→ Canvas security issue, check CORS settings

Error: "Permission denied"
→ Browser blocked download, check settings

If Network Tab shows "Failed" for CDN:
→ Firewall/antivirus blocking CDN
→ No internet connection
→ Corporate network restrictions
